swp_stack_trace 原理图解

信号处理 . 栈展开 . libunwind . 离线解析

1. 信号处理全流程

内核态 swp_stack_trace 安全出口 不推荐

2. 栈帧与展开原理

每次函数调用分配一个栈帧。libunwind 沿 DWARF unwind table 逐帧回溯返回地址。

3. libunwind 工作流程

unw_step 返回正值 = 还有上一帧; 返回 0 = 栈顶。全程不调 malloc/printf。

4. async-signal-safe 对比

信号 handler 被中断的代码可能持有锁。handler 中再次拿同一把锁 = 死锁。

5. Core dump 与堆栈互补

先打印堆栈文本（原始现场），再 re-raise 生成 core dump（完整内存）。

6. 离线解析流程

用法: ./run.sh 2>&1 | ./symbolize_stack.py binary 或 ./symbolize_stack.py logfile
